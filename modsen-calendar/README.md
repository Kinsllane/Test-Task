## Техническое задание

Необходимо реализовать React приложение "Modsen Calendar" по предоставленному дизайну из Figma. Пользователь сможет добавлять собственные события, искать мероприятия через внешний API и сохранять расписание локально.


## API
Для поиска мероприятий необходимо интегрировать внешний API. Например, можно использовать [API KudaGo](https://docs.kudago.com/api/#), который предоставляет информацию о мероприятиях. 

## Необходимый функционал:
- Календарь с отображением всех событий;
- Возможность добавлять, редактировать и удалять события;
- Функция поиска названий мероприятий через API;
- Сохранение всех данных локально;
- Отображение избранных событий в отдельной секции;
- Drag-and-Drop функциональность.

### Дополнительные указания:

- Реализация Loader для отображения запасного UI при подгрузке данных;
- Использование любых библиотек для стилей, валидации и так далее - запрещены, кроме тех, которые рекомендуемы в пункте “Используемые технологии”.

## Описание экранов
1. Страница [Calendar](https://www.figma.com/design/v1mVzZ2UutRwO9q93Kt1Tv/Modsen-Calendar?node-id=0-1&p=f&t=UXWnlzJ5EterkM9d-0)

Приложение должно предоставлять пользователю календарь (вид по дням и неделям). События можно добавлять по времени с 00:00 по 23:00. В календарь можно добавлять события, заполнив специальную [форму](https://www.figma.com/design/v1mVzZ2UutRwO9q93Kt1Tv/Modsen-Calendar?node-id=1-12640&t=UXWnlzJ5EterkM9d-0). Открыть ее можно, нажав в области поля календаря. Если форма на создание события открыта, но поля не заполнены, то кнопка "Save" не активна и пользовать может закрыть окно при нажатии вне его области. Обязательными полями при создании события являются: название события, место проведения, дата и продолжительность. 

В случае, если обязательные поля не заполнены, то кнопка "Save" не доступна и срабатывает валидация необходимых полей: максимальная длина полей - 20 символов, формат даты: ДД.ММ.ГГГГ., формат продолжительности события: ЧЧ:ММ-ЧЧ:ММ. Для заполнения даты и продолжительности времени события можно использовать маску или placeholder.

Пользователь может перетаскивать событие из одной даты в другую. Когда пользователь кликает на событие и начинает его перетаскивать, оно становится "перемещаемым" и может быть перенесено в другую ячейку календаря, которая представляет собой другой день или время.
Когда событие отпускается в новой ячейке календаря (например, в другом дне или времени), приложение должно обновить дату и время события. 
После того, как событие перемещено, календарь должен отобразить новое расположение события в соответствующем месте, и пользователь увидит, что дата или время события изменились.

При клике на существуещее событие пользователь может изменить данные созданного события(при этом открывается то же модальное окно, которое появлятся при создании события, только в этом случае поля уже будут заполнены информацией события и ее можно будет изменять. При этом можно менять только название встречи. Также событие можно удалить, нажав на кнопку "Delete", при этом пользователь должен подтвердить свои действия для удаления).

Если Вы не выполняете дополнительные задания, то некоторые части макета Вам необходимо сделать статичными.


2. Страница [New Event](https://www.figma.com/design/v1mVzZ2UutRwO9q93Kt1Tv/Modsen-Calendar?node-id=0-1&p=f&t=UXWnlzJ5EterkM9d-0)

Форма добавления нового события отображается при клике на сетку календаря. Пользователь вводит название события, локацию, дату, время и краткое описание. Добавленные события сразу отображаются в соответствующей ячейке календаря, также может выбрать цвет события(список цветов доступен при открытия списка с различными цветами(их можно использовать в списке до 7-ми), далее этот цвет отображает границы самого события).
В приложении есть возможность искать внешние события через API (например, использовать KudaGo API или любой другой открытый API с информацией о мероприятиях). 

Пользователь вводит ключевое слово (например, «yoga» или «conference»), получает список подходящих мероприятий и выбирает понравившиеся(всего событий в списке может быть до пяти). При клике на событие из выпадающего списка, оно отображается в поле ввода и список больше не отображается, также его можно закрыть при клике вне его области. Для того, чтобы при вводе запросов на сервер было меньше,  можно использовать debounce.

Все данные расписания должны сохраняться в LocalStorage. Это позволяет пользователю не терять расписание при перезагрузке страницы или закрытии браузера. (Хранилище также используется для сохранения избранных событий.)

3. Дополнительная страница [Registration](https://www.figma.com/design/v1mVzZ2UutRwO9q93Kt1Tv/Modsen-Calendar?node-id=0-1&p=f&t=UXWnlzJ5EterkM9d-0)

Для реализации регистрации пользователя используйте Firebase и регистрацию по электронной почте.
Форма регистрации должна содержать поля: имя, фамилия, электронная почта, пароль и подтверждение пароля. Имя и фамилия должны быть длиной от 2 до 50 символов и состоять только из латинских букв. Электронная почта должна соответствовать формату example@domain.com и быть уникальной. Пароль должен быть длиной не менее 8 символов, содержать одну заглавную букву, одну строчную, одну цифру и один специальный символ. Подтверждение пароля должно совпадать с полем "Password". Пользователь должен иметь возможность показывать и скрывать пароль через иконку рядом с полем. Ошибки ввода отображаются сразу при заполнении формы. Кнопка регистрации становится активной только при корректном заполнении всех полей. При успешной регистрации выводится сообщение и пользователь переходит на страницу календаря.

Также отображаются его актуальные данные: почта и имя, также есть возможность выйти из аккаунт с подтверждением действий пользователя.

4. Дополнительная страница [Authorization](https://www.figma.com/design/v1mVzZ2UutRwO9q93Kt1Tv/Modsen-Calendar?node-id=0-1&p=f&t=UXWnlzJ5EterkM9d-0)

Если пользователь зарегистрирован и позже вышел из аккаунта, он имеет возможность опять зайти по своей существующей учетной записи. Для этого в форме необходимо ввести свое имя и пароль. В случае, если такого пользователя не существует, то необходимо уведомить пользователя, для этого можно отоюражать тост с ошибкой, который будет исчезащь через несколько секунд на странице. Также должна срабатывать валидация при вводе значений в полях ввода, она будет такая же как и в форме регистрации у соответствующих полей.


## Используемые технологии

### Для react

- ***node.js*** - программная платформа, основанная на движке V8 (транслирующем JavaScript в машинный код);
- ***typescript*** - библиотека, добавляющая статическую типизацию;
- ***react*** - JavaScript-библиотека для создания пользовательских интерфейсов;
- ***redux*** - инструмент для управления состоянием данных и пользовательским интерфейсом в приложениях;
- ***SCSS*** - препроцессор, позволяющий писать код для стилей CSS;
- ***vite*** -  сборщик модулей, который позволяет скомпилировать JavaScript-модули в единый JS-файл;
- ***firebase*** - набор инструментов и сервисов для разработки мобильных и веб-приложений от Google(используем в случае работы с дополнительными задачами);
- ***babel*** - транспайлер, преобразующий код из одного стандарта в другой;
- ***husky*** - современное решение для управления Git хуками;
- ***eslint*** - линтер для JavaScript кода;
- ***prettier*** - инструмент для автоформатирования кода;
- ***axios*** - библиотека для создания асинхронных запросов;
- ***commitlint*** - инструмент, который проверяет сообщения commit-ов на соответствие общепринятым стандартам их описаний;
- ***yarn*** - менеджер пакетов.
